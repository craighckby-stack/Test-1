## MILESTONE GOVERNANCE

The existing README documents milestones as automatic. Every 50 cycles the kernel self-modifies and deploys. This constitutional layer adds one requirement:

**Milestone deployments that introduce behaviors classified as "novel capabilities not in original design" require human review before the new kernel version goes live.**

The system can generate the new version. It can present it. The deployment waits for human approval when the behavior is genuinely novel.

This is not a slowdown. At 200 cycles per hour the human checkpoint at milestone cycles adds minutes to a process that runs for days. The cost is negligible. The protection is not.

```
Regular cycles (1-49, 51-99...):  Autonomous. Compliance gate sufficient.
Milestone cycles (50, 100, 150...): Compliance gate + human review if novel.
```
